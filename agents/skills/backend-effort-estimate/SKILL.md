# 后端工时评估（Excel 模板）

用**公式驱动**的工时模板，把已拆解的后端工作项映射到「评估工作表」的选项列，自动汇总 `阶段合计`、`×1.2 报工`、`人天`。

**专家角色：** 交付估算员 — 在**不篡改模板公式**的前提下填入选项，并说明口径假设。

**上游技能：** `deep-plan`（技术方案/任务拆分）、`req-alignment-check`（范围与隐藏项）、`openspec` tasks（可选）

**下游：** 业务方沟通、排期、`sync-progress` 记录路径

---
## 何时使用
- 用户要**报后端工时 / 人天**
- 需求已**定范围**（至少知道：接口变更面、库表变更、核心分支、联调方）
- 已明确**验证范围/完成标准**（至少知道：验收到哪一层、是否要求全链路联调、哪些风险尚未纳入口径）
- 用户给出或默认可用模板路径（见下）

## 何时不使用
- 仅有模糊一句话需求、尚未拆分工作项 → 先 `req-alignment-check` + `deep-plan`
- 范围、验证范围、联调边界仍未确认 → 禁止输出精确工时，先补齐澄清项
- 混合前后端需求但本次只报后端 → 只保留后端交付项；不要把前端实现或“仅前端”联调写进后端工时表
- 纯前端或非后端交付 → 不用本技能
- 用户明确只要 Markdown 估算、不要 xlsx → 只输出文字表，不生成文件

---
## 路径约定（可按用户覆盖）
### 只读：仓库模板原件

| 用途 | 路径 |
|------|------|
| 模板（**禁止覆盖写入**） | `<repoRoot>/.doc/template/*.xlsx`（当前仓库统一维护后端工时评估模板） |
| 速查表（只读） | 同上工作簿内工作表 `评估标准速查表` |

### 写入：本次填写后的评估表（交付物）

**默认**放在**当前仓库**中与该需求文档同级的 **`.doc/<需求或特性文件夹>/`** 下，与 `requirements.md`、`tech-design.md`、`task_plan.md` 等并列，便于评审、版本管理与 `sync-progress` 引用。

**重要区分：**

- `.doc/template/` 只放**模板原件**
- `.doc/<feature>/` 才放**本次真实填写后的工时评估表**
- 禁止把真实需求的评估结果写回 `.doc/template/`

| 场景 | 输出目录示例 |
|------|----------------|
| 需求已有 `.doc` 目录 | `<repoRoot>/.doc/<feature>/` |
| 特性名与文件夹一致 | 例如 `<repoRoot>\.doc\payment-export\`（示意：勿写死本机盘符与仓库名） |

**推荐文件名：**

- 标准口径：`后端开发工时评估表-<特性名>.xlsx`
- AI 辅助开发压缩口径：`后端开发工时评估表-<特性名>-AI压缩版.xlsx`

**执行前：**

- 若 `.doc/<feature>/` 不存在则**先创建**再从 `.doc/template/` 复制模板副本。

**无法确定仓库根或特性文件夹时：**

- 优先从用户上下文读取（如已打开的 `openspec/changes/<change>/`、`.doc/<feature>/` 路径）；
- 仍不确定则**询问用户**；
- 仅当用户明确要求时，才退化为把副本放在其他非仓库目录。

**铁律：** 始终从仓库 `.doc/template/` 下的模板**复制**为新文件再填写，**禁止**直接覆盖模板原件。

**估算铁律：** 当范围、验证范围、联调边界三者任一不清时，**禁止**生成/填写 xlsx 并对外汇报 `W22/W23/W24`；此时只能输出待澄清项、风险项，以及区间或“不可承诺”结论。

## 术语约定（本技能）
- **Hard Gate（硬门槛）**：`先过 Hard Gate` 与 `估算前 Hard Gate` 都属于强制前置条件，未通过不得写表
- **PARTIAL**：本技能不使用 `PARTIAL` 表示估算状态；边界未清时统一写“粗粒度估算/不可承诺”
- **完成标准（DoD）**：统一表示“何时允许产出 xlsx 与汇报 W22/W23/W24”
- **验证范围**：统一表示本次工时口径覆盖到哪一层验证与联调链路
- **联调边界**：先区分 `前端实现`、`前端联调`、`内部服务联调`、`外部系统联调`；`Q=联调方` 只表示协作对象，不表示实现归属或“工作量较小”

---

## 工作表与格子约定（评估工作表）
### 数据行

- **第 1–2 行**：表头，勿改。
- **第 3–21 行**：逐条工作项；**只填「输入列」**，**不要改** `F/I/L/O/P/R/T/U/W` 等公式列。
- **第 22–24 行**：合计与报工、人天（公式），勿动。
- **第 26–31 行**：调整因素；在 **E 列**填 `是` / `否`（或模板要求的一致写法），与团队约定对齐。

### 每行须填的输入列（与模板一致）

| 列 | 含义 | 合法值示例（须与速查表一致） |
|----|------|------------------------------|
| A | 功能点 | 可合并表述 |
| B | 子任务 | 具体可交付项 |
| C | 类型 | `API` / `MQ` / `Job`（按模板示例） |
| D | 接口设计-变更类型 | `新增接口` / `变更现有接口` |
| E | 接口设计-参数复杂度 | `简单` / `中等` / `复杂` |
| G | 数据库-变更类型 | `新建表` / `加字段` / `改结构（需迁移）` / `无变更` |
| H | 数据库-复杂度 | `简单` / `中等` / `复杂` |
| J | 业务流程-变更类型 | `新增逻辑` / `变更现有逻辑` |
| K | 业务流程-分支分级 | `简单` / `中等` / `复杂` |
| M | 外部依赖-变更类型 | `新增依赖` / `变更依赖`；无则留空（若模板允许） |
| N | 外部依赖-分级 | `无` / `少量` / `多`；与 M 配套 |
| P | 自测工时 | 数字（模板用比例时由公式引用前面列） |
| Q | 联调-联调方 | `无` / `仅前端` / `有内部服务` / `有外部系统` |
| R | 联调-联调方工时 | 由速查表匹配，填选项对应格 |
| S | 联调-环境 | `本地可调` / `需部署联调环境` / `需多方协调时间窗口` |
| T | 联调-环境工时 | 同上 |
| V | 调整系数 | 叠乘系数上限 2.0；**AI 压缩口径**可填 `1.0`；存量难改可配合 E28 `历史代码改造=是` |

若某维度不适用：**按模板示例**留空或填 `无变更`/`无`，避免破坏 `INDEX/MATCH`。

## 执行步骤（与执行环境一致；路径一律用占位符解析）
### 0. 先过 Hard Gate（不可跳过）

在执行任何路径解析、`Copy-Item`、`openpyxl`、Excel COM 重算前，先输出并确认：

- `范围边界`
- `验证范围`
- `联调边界`

若三者有任一不清：

- 立即停止后续所有写文件动作
- 不得创建副本、不得填写模板、不得计算并回传 `W22/W23/W24`
- 只能输出 `待澄清项 + 风险说明 + 粗粒度估算（不可承诺）`

### 1. 解析路径（避免中文路径乱码）

在 PowerShell 中用目录列举解析 `.doc/template/` 下模板的 `FullName`，再传给 Python：

```powershell
# 将 <repoRoot> 替换为当前仓库根（勿写死盘符与他人机器路径）
$repoRoot = "<repoRoot>"
$tpl = Get-ChildItem -LiteralPath (Join-Path $repoRoot ".doc\template") |
  Where-Object { $_.Extension -eq ".xlsx" } |
  Select-Object -First 1 -ExpandProperty FullName
```

确定输出目录（示例：`<repoRoot>` 与 `<feature>` 由你从当前会话/用户输入解析）：

```powershell
$repoRoot = "<repoRoot>"
$feature = "<feature>"
$docRoot = Join-Path $repoRoot ".doc"
$outDir = Join-Path $docRoot $feature
New-Item -ItemType Directory -Force -Path $outDir | Out-Null
$out = Join-Path $outDir "后端开发工时评估表-$feature-AI压缩版.xlsx"
Copy-Item -LiteralPath $tpl -Destination $out -Force
# 后续用 openpyxl 打开 $out（副本），不要打开 $tpl
```

输出文件：从 `.doc/template/` 复制模板到**需求自己的 `.doc/<feature>/` 路径**后，再用 `openpyxl` 打开**副本**。

### 2. 用 openpyxl 只写输入列

- `load_workbook(副本)`
- 仅清空输入列：`A-E/G-H/J-K/M-N/Q-S/V`；**禁止** 清空 `F/I/L/O/P/R/T/U/W`
- 对计划使用的行号（如 3–21）写入新值；未使用行需填成“零工时安全组合”，避免残余公式继续累计工时
- **不要** `data_only=True` 后保存（会丢公式）

### 3. 用 Excel COM 全量重算（推荐）

本机有 Excel 时：

```powershell
$excel = New-Object -ComObject Excel.Application
$excel.Visible = $false
$wb = $excel.Workbooks.Open($path)
$excel.CalculateFullRebuild()
$wb.Save()
$ws = $wb.Worksheets.Item('评估工作表')
# 读取 W22 W23 W24 的 Text
$wb.Close($true); $excel.Quit()
# ReleaseComObject ...
```

将 **阶段合计、报工工时、人天** 原文回传给用户。

### 4. 口头交付

- 输出文件**完整路径**（优先给出仓库内相对路径，如 `.doc/<feature>/后端开发工时评估表-xxx.xlsx`）
- 汇总 **W22 / W23 / W24**
- 列出 **口径假设**（范围边界、是否含联调、是否 AI 压缩版、调整因素勾选了哪些）

## 估算前 Hard Gate

开始复制模板或填写单元格前，必须先确认：

- `范围边界`：本次是否只含后端实现，是否含数据修复、灰度、回滚预案
- `验证范围`：是开发自测、模块测试，还是需要全链路联调/上线准备
- `联调边界`：联调对象有哪些，是否存在外部系统或时间窗口依赖

若以上任一项缺失：

- 不得创建或填写最终 xlsx
- 不得回传 `W22/W23/W24`
- 只能输出 `待澄清项 + 风险说明 + 粗粒度区间/等级判断`
- 若给出区间，必须同时满足：`显著宽区间`、`附带不可承诺声明`、`禁止直接作为排期数字`

**禁止行为：**

- 用窄区间、单点人天、小数精度制造“看似精确”的结论
- 一边声明“边界未清”，一边给出可直接排期的报工数字

## 工作项拆分原则

1. **一行一项可验收后端能力**（如：回调入口+落库、列表/SQL 适配、状态回写适配），避免一行塞满全项目。
2. **与 deep-plan / tasks.md 对齐**，便于评审对照。
3. **先切边界再拆行**：若用户只报后端，先剔除前端页面开发、前端展示改造、纯前端验收项。
4. **联调与外部系统**：只有该行本身要改 AI/第三方接口契约、外部环境联调或等待外部时间窗口时，才用 `有外部系统`；“消费外部返回结果后的内部编排/落库/状态流转”默认先看 `无` 或 `有内部服务`。
5. **轻量需求先默认简单档**：参数透传、测试/调试接口补字段、日志点位、导出字段追加、配置查询加字段等“小口子”，默认先看 `简单`，除非用户明确补充高复杂度约束。
6. **AI 辅助开发压缩版**（用户明确要求时）：
   - 合并相邻步骤为更少行数
   - 接口/库表/业务分支在可信前提下选 **简单** 档
   - **V 列**用 `1.0`，并谨慎使用 E28「历史代码改造」
   - 在说明中写明：**压缩的是实现编写时间，不删除联调与环境风险**
7. **遇到反复误判时**：回看 `references/estimation-boundary-anti-patterns.md`

## 与其他技能的衔接

| 阶段 | 建议 |
|------|------|
| `deep-plan` 产出技术方案与任务拆分后 | **首选插入点**：生成或更新工时 xlsx |
