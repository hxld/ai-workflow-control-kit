# 审查反模式清单

## Anti-Patterns 表

| ❌ Don't | ✅ Do Instead | 级别 |
|----------|---------------|------|
| 空 catch 块吞异常 | 至少 log.error + 上下文 | P1 |
| 魔法数字硬编码 | 提取命名常量 | P3 |
| N+1 查询循环 | 批量查询或 JOIN | P2 |
| 硬编码密钥/令牌 | 环境变量 + 密钥管理 | P0 |
| 不处理 null/undefined | 卫语句 + Optional | P1 |
| 重复代码块 | 提取共享方法/模块 | P2 |
| 长方法 > 50 行 | 提取方法，单一职责 | P3 |
| 嵌套 > 3 层 | 卫语句 + 提前返回 | P2 |
| 静默失败（无日志）| 记录错误 + 传播 | P1 |
| 全局可变状态 | 依赖注入 + 局部作用域 | P2 |

## 代码异味

| 异味 | 检测 | 修复 |
|------|------|------|
| 长方法 | > 50 行 | 提取方法 |
| 参数过多 | > 4 个 | 参数对象 |
| 嵌套过深 | > 3 层 | 卫语句 |
| 重复代码 | 相似块 | 提取共享方法 |
| 魔法数字 | 未命名常量 | 命名常量 |

## 安全检查重点

- XSS/注入（SQL/NoSQL/命令/GraphQL）/SSRF/路径遍历
- AuthZ/AuthN 缺口、缺少租户检查
- 密钥泄露、API key 在日志/env/文件中
- 竞态条件：check-then-act、TOCTOU、缺少锁
- 不安全反序列化、弱加密、不安全默认值

## 测试与 Mock 反模式

| 反模式 | Review 信号 | 严重度 |
|--------|-------------|--------|
| Mock 被测行为 | mock 返回目标方法应计算出的结果，断言只检查 mock 被调用 | P1 |
| Test-only 生产入口 | 为测试暴露 public helper、setter、debug flag 或绕过校验路径 | P1 |
| 不理解 contract 就 mock | mock 未说明真实依赖的异常、空值、幂等、重试或权限语义 | P1 |
| 不完整 mock | happy path 绿，失败分支、must-not、副作用、状态推进无断言 | P2 |
| 集成验证缺失 | mapper、序列化、事务、权限、消息、异步链路从未被真实验证 | P1/P2 |

**Review 处理：** 低歧义 mock 误用可归为 `AUTO-FIX`；若需要重划测试层级、补 contract 或改变可测性设计，归为 `ASK`。

## 反馈处理分流

| 分流 | 适用 | Review 建议 |
|------|------|-------------|
| `AUTO-FIX` | 局部、确定、可复跑验证，不改变业务语义 | 标注建议命令与验证范围 |
| `ASK` | 需要业务 owner、架构取舍、scope 扩张或迁移判断 | 集中到 Next Steps，不分散追问 |
| `PUSH-BACK` | reviewer 假设错误、会破坏现有 contract、与需求冲突 | 给出证据和替代方案 |
