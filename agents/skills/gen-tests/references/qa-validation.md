# QA Validation Pipeline

## Test-Fix-Verify 三阶段循环

| 阶段 | 动作 | 说明 |
|------|------|------|
| Test | 运行完整测试套件 | 捕获所有失败 |
| Fix | 分类并修复每个失败 | 见故障分类表 |
| Verify | 仅重跑受影响测试 | 不跑全量，加快反馈 |

## 故障分类与处理

| 分类 | 判断依据 | 处理方式 |
|------|----------|----------|
| 测试代码 bug | 断言逻辑错 / mock 配置错 | 直接修复测试代码 |
| 源代码 bug | 业务逻辑不符合预期 | 根因分析 → 最小修复 → 验证 |
| 环境问题 | 依赖缺失 / 配置不匹配 | 尝试自动修复 → 失败则降级 |

## 轮次追踪

| Round | Tests | Fixed | Status |
|-------|-------|-------|--------|
| 1 | N | N | pass / fail / pause |

**边界：** 最多 3 轮。3 轮后仍失败 → PAUSE + 输出诊断摘要。

## 降级表达规范

降级时必须显式标注：
- `evidence gap: [缺失什么证据]`
- `降级原因: [环境/依赖/配置问题]`
- 禁止裸写 "测试失败" 或 "跳过测试"，必须说明具体原因

## Mock 规则

| 依赖 | Mock? |
|------|:-----:|
| DB/API/外部 | 是 |
| 内部逻辑 | 否 |
| 纯函数 | 从不 |

## 场景覆盖（必须）

| 场景 | 测试 |
|------|------|
| 正常路径 | 正常输入，成功 |
| 空/Null | 空列表/字符串/null |
| 无效 | 错误格式/缺失数据 |
| 边界 | 0, MAX, MIN |
| 异常 | 预期异常 |

## 外部系统集成追加场景

- 契约匹配：输入/输出 schema 是否稳定解析
- 幂等：重复请求是否重复入库/重复展示
- 状态流转：状态机推进/回退是否正确
- 回归：既有接口行为不被破坏
- 集成验证项：鉴权、真实报文、网络异常标记为手工验证
