# Verification Helpers

## 常见声明 vs 所需证据

| 声明 | 需要什么证据 |
|------|-------------|
| "测试通过" | 测试命令输出（0 失败） |
| "编译成功" | `mvn compile` 输出（BUILD SUCCESS） |
| "代码正确" | Lint 结果、类型检查结果 |
| "已修复" | 具体改动 diff、复测通过 |
| "无问题" | 检查命令执行结果、对比前后状态 |
| "文档更新" | 文件 diff 显示新增/修改内容 |

## 红红旗词汇

当审查代码或输出时，注意以下警告信号：

| 词汇 | 问题 | 行动 |
|------|------|------|
| "应该"、"大概"、"可能" | 模糊声明，不是确定知识 | 要求提供具体证据 |
| "基本上"、"差不多"、"几乎" | 缺乏精确性 | 重新验证 |
| "没问题"、"Great!"、"完美" | 过早乐观 | 贴出验证命令输出 |
| "我觉得"、"我认为" | 主观判断 | 要求客观证据 |
